Introduction to Workflows

What are Workflows?

Workflows are macro-level strategic maps that structure project management, strategies (like marketing, development), manufacturing, and other pipelines. They are built on top of templates and populated with actual content and requirements.

Key Concept: Workflows take the structure from templates and fill it with real-world content, requirements, and data.

Workflow Capabilities

  • Four creation methods - From template, empty, copy, or linked
  • Dual modes - Creation mode for building, Execution mode for tracking
  • Rich attachments - Links, images, notes, and comments
  • Workflow linking - Synchronize structure across multiple workflows
  • Export to boards - Transform workflows into execution spaces

Four Creation Methods

Method 1: From Template

Create a workflow based on a predefined template structure. This is the recommended method for structured workflows.

1

Click "New" button and select "From Template"

2

Enter workflow name and select a template

3

Click "Create Workflow" to start building

Method 2: Empty Workflow

Create a simple 1-level workflow instantly without needing a template. Perfect for quick checklists or simple tracking.

Method 3: Copy Existing

Duplicate an existing workflow with all its structure and content. Great for creating variations of existing workflows.

Method 4: Linked Workflow

Create a workflow that synchronizes its structure with other workflows. Perfect for tracking the same requirements across multiple teams or departments.

Creation vs Execution Modes

Understanding the Two Modes

Creation Mode

Purpose: Build and modify workflow structure

  • Add/edit/delete units
  • Change names and descriptions
  • Modify structure
  • Add attachments

Execution Mode

Purpose: Track progress and completion

  • Check off "Done" checkboxes
  • Enter/update grades
  • View progress bars
  • Filter by tags
Tip: Always save your work before switching modes! Changes in one mode don't automatically save to the other.

Building Workflow Structure

Adding Units

In Creation mode, you can add units at any level of your workflow hierarchy. The button name changes based on your template (e.g., "+ Add Rule", "+ Add Control").

1

Click the Add Button

Look for the "+ Add [Level Name]" button at the appropriate level

2

Fill in Details

Enter the unit name, description, and any other properties

3

Add Child Units

Click the + button on a parent unit to add children

4

Save Structure

Click "Save Structure" to persist your changes

Attachments and Content

Four Types of Attachments

Links

External resources and documents

Images

Screenshots and diagrams

Notes

Rich text with formatting

Comments

Timestamped discussions

Remember: Attachments are part of the workflow structure, so click "Save Structure" after adding or modifying them!

Workflow Linking

Synchronized Workflows

Linked workflows share the same structural changes. When you modify the structure of one linked workflow, those changes automatically propagate to all other linked workflows in the group.

Important: Only structural changes are synchronized. Execution state (checkmarks, grades) remains independent for each workflow.

Use Cases for Linking

  • Multi-department compliance - Same requirements across departments
  • Multi-region operations - Same structure, different regions
  • Multi-team projects - Synchronized task lists

Exporting to Boards

Transform Workflows into Boards

Export workflows to boards to create execution spaces where teams can manage tasks, track progress, and collaborate.

1

Select Export Option

In Execution mode, click "Export to Board" or use the board button on a unit

2

Choose Export Type

Export entire workflow, specific unit tree, or by tags

3

Configure Board

Set board name, columns, and initial settings

4

Start Executing

Navigate to the board and start managing tasks
