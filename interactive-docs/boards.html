Introduction to Boards

What are Boards?

Boards are micro-level execution spaces where workflows land for task management. They provide a Kanban-style interface for organizing, tracking, and completing tasks that originated from workflow structures.

Key Concept: Boards are where consultants maintain control over dynamic lists and references, while clients manage the workspace for task execution.

Board Features

  • Kanban columns - Organize tasks by workflow stages
  • Dynamic lists - Reference tree structures from workflows
  • Milestones - Time-based or completion-based project phases
  • Categories - Organizational buckets (departments, types)
  • Groups - Cross-cutting collections for bulk operations
  • Assignments - Task ownership and scheduling

Working with Columns

Column Management

Columns represent the stages of your workflow. You can customize columns to match your process, add new ones, reorder them, and configure their properties.

1

Default Columns

New boards start with standard columns: To Do, In Progress, Review, Done

2

Add Custom Columns

Click the "+" button to add columns that match your workflow

3

Reorder Columns

Drag and drop columns to reorder them

4

Configure Properties

Set column colors, limits, and behaviors

Tip: The "References" column is special - it displays the dynamic list tree structure from the exported workflow. Consultants typically manage this column.

Categories

Organizational Grouping

Categories allow you to organize tasks by departments, types, or any other organizational structure. They're perfect for filtering and reporting.

Common Category Types

  • Departments - IT, Finance, Legal, HR
  • Task Types - Technical, Administrative, Documentation
  • Priority - Critical, High, Medium, Low
  • Regulations - GDPR, ISO 27001, NIST CSF

Dynamic Lists

Reference Tree Structures

Dynamic lists display the hierarchical structure from your exported workflow. They serve as reference material that consultants typically manage, while clients use the workspace area for task execution.

Important: Dynamic lists are typically locked for editing by clients. Consultants maintain control over the reference structure to ensure compliance requirements remain accurate.
