### 3. USE CASE: COMPLIANCE MANAGEMENT CONSULTANCY

**3.1 The Challenge**
- Complexity of regulatory compliance
- Different regulations, different structures
- Client diversity and customization needs
- Maintaining consistency across engagements

**3.2 The Newflows Solution**
**Phase 1: Consultant Creates Templates**
- Design regulation-specific structures
- Define compliance requirement levels
- Configure data collection points
- Build reusable frameworks

**Phase 2: Consultant Creates Workflows**
- Apply template to specific regulation
- Build comprehensive compliance map
- Document requirements hierarchically
- Share as reference guideline with clients

**Phase 3: Export to Boards for Execution**
- Transfer workflow to execution environment
- Consultant manages references and structure
- Client executes compliance tasks
- Track progress and completion

**3.3 Real-World Example: NIST CSF Compliance**
**Template Design:**
- Level 1: Functions (Identify, Protect, Detect, Respond, Recover)
- Level 2: Categories (Asset Management, Access Control, etc.)
- Level 3: Subcategories (specific requirements)
- Level 4: Implementation Actions

**Workflow Creation:**
- Map NIST CSF requirements
- Add documentation references
- Include audit evidence requirements
- Tag priority items

**Board Execution:**
- Dynamic list shows full structure
- Tasks created for each action
- Milestones for each function
- Categories for different departments
- Progress tracking per category

### 10. PLATFORM TERMINOLOGY

**Templates:** Pre-configured structures that define workflow capabilities
**Workflows:** Macro-level strategic maps of processes or compliance requirements
**Boards:** Micro-level execution spaces for task management
**Levels:** Hierarchical tiers in a template or workflow structure
**Units:** Individual items at any level of the hierarchy
**Dynamic Lists:** Tree-structured reference materials in boards
**References:** Locked reference materials that inform execution
**Milestones:** Time-based or completion-based project phases
**Categories:** Organizational buckets for tasks (e.g., departments)
**Groups:** Cross-cutting collections of tasks for bulk operations
